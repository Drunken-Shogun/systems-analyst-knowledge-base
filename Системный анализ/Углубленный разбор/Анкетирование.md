> Если существует *ряд одинаковых вопросов к широкой группе лиц* или *не хватает времени для прямой коммуникации со всеми сторонами*, в таком случае можно применить анкетирование.
> Данный метод также используется для проведения статистических исследований с пользователями. 

## Как провести анкетирование?
1. Поставьте ясную и конкретную цель
2. Выберите целевую группу опрашиваемых (учитывайте репрезентативность)
3. Собрать контакты для рассылки анкеты
4. Получить согласие на участие
5. Обозначить сроки ответа
6. Подготовить анкету

>[!abstract] Готовим анкету
>**Шаблон**:
>- Вводная часть - в ней раскройте контекст и цель анкеты
>- Список вопросов
>- Завершающие слова

>[!tip] Best practices
>- В начале анкеты поместите простые и нейтральные вопросы, для разогрева
>- Для проверки честности ответа можно использовать один вопрос в разных формах
>- Слова в опросе должны быть понятны людям
>- Следите за тем, чтобы логика вопросов и ответов не была поломана формулировкой (использование большого количества *не*)
>- Ботайте *статистику* для обработки результатов

Собираем требования из открытых вопросов, проводим статистический анализ закрытых вопросов, утверждаем требования у ответственных -> *BINGO*

**Плюсы**:
1. Широкий выбор респондентов
2. Малые временные затраты
3. Возможность использовать результаты в будущем
4. Это основа для статистических исследований
5. Открытые вопросы - миниинтервью

**Минусы**:
1. Не выявляет "скрытые" требования
2. Важно уметь в *статистику*
3. Может не оказаться нужного числа респондентов
4. Чем больше открытых вопросов, тем больше времени для анализа и меньше респондентов
5. Сами по себе редко закрывают все потребности
